Unsupervised learning is different from most machine learning because there is no ground truth label or target value to compare the model's output against.

# Hard vs. Soft Clustering

**Hard Clustering, e.g. K-Means:** Absolute boundaries. Every data point belongs to exactly one cluster. Mathematically: $P(C_k|X_i) \in \{0,1\}$.

**Soft Clustering, e.g. GMM:** Probabilistic boundaries. Data points have a probability distribution over all clusters, such as 80% to Cluster A and 20% to Cluster B. Mathematically: $P(C_k|X_i) \in [0,1]$.

# Clustering Families

**1. Centroid-Based (Partitioning) Clustering**
- **How it works:** Finds a central point (centroid) for each cluster and groups data based on the shortest distance to these centers. Assumes clusters are spherical and roughly the same size.
- **Algorithms:** K-Means.
- **Assignment:** Hard.

**2. Distribution-Based (Probabilistic) Clustering**
- **How it works:** Assumes the data was generated by a mix of statistical distributions (e.g., Gaussian/Normal distributions). Captures the center, spread, and orientation (elliptical shapes) of the data.
- **Algorithms:** Gaussian Mixture Models (GMM).
- **Assignment:** Soft.

**3. Hierarchical Clustering**
- **How it works:** Builds a tree of clusters (a dendrogram) by iteratively merging small clusters together (Bottom-Up / Agglomerative) or splitting a giant cluster apart (Top-Down / Divisive). Doesn't require guessing the number of clusters in advance.
- **Algorithms:** Agglomerative Clustering (using linkages like Ward, Single, Complete).
- **Assignment:** Hard.

**4. Density-Based Clustering**
- **How it works:** Looks for continuous regions where data points are packed tightly together, separated by empty space. Excellent for finding non-linear shapes (like the "Moons" dataset) and automatically flags isolated points as noise/outliers.
- **Algorithms:** DBSCAN, HDBSCAN.
- **Assignment:** Hard.

# K-Means Model

**Category:** Centroid-Based (Partitioning) Clustering

**Video:** [K-Means explained](https://www.youtube.com/watch?v=4b5d3muPQmA&pp=ygUTc3RhdHMgcXVlc3Qgay1tZWFucw%3D%3D)

## How the Model Works

**Initialization:** You start by defining the number of clusters you want to identify, which represents the K. The algorithm then randomly selects K distinct data points to act as the initial cluster centers.

**Assignment:** The algorithm measures the distance from every data point to the initial cluster centers and assigns each point to the cluster that is closest to it.

**Updating:** Once all points are grouped, it calculates the mean of each newly formed cluster to establish new cluster centers.

**Iteration:** It repeats the process of measuring distances and re-assigning points to the nearest mean values until the clusters stop changing.

## The Math Behind It

K-means uses Euclidean distance to assign points to the nearest cluster center: $d(x, c) = \sqrt{\sum_{j=1}^{p}(x_j - c_j)^2}$.

Because it depends on distance, features should be scaled. It also suffers from the curse of dimensionality: in high dimensions, distances become less informative.

## Useful Metrics

K-means commonly uses both Inertia, also called WCSS, and the Silhouette Score.

## K-Means Hyperparameters (Scikit-learn)

**n_clusters ($K$):** The number of clusters you want the algorithm to find, which must be predefined.

**n_init:** The number of times the algorithm runs from scratch with different random starting points. The model ultimately keeps the run that produces the lowest Inertia.

**max_iter:** The maximum number of steps, or iterations, a single run will take to try and reach convergence before it is forced to stop.

## Limitations

**Pre-defining K:** K-means cannot choose the optimal number of clusters; users must test different values of K.

**Sensitivity to Starting Points:** Random initial centers can lead to poor clusters, so the algorithm is usually run multiple times and keeps the lowest-variance result.

## Biases

K-means has two main biases:

1. The spherical assumption
2. The convex assumption

# Gaussian Mixture Models (GMM)

**Category:** Distribution-Based (Probabilistic) Clustering

**Video:** [Gaussian Mixture Models explained](https://www.youtube.com/watch?v=EWd1xRkyEog)

A probabilistic, generative alternative to K-means. It assumes data is generated by a mixture of Gaussian distributions, allowing more flexible cluster shapes.

## How the Model Works

**Soft Clustering:** Points are assigned probabilities across clusters, such as 80% Cluster A and 20% Cluster B.

**Optimization (EM Algorithm):** GMM uses Expectation-Maximization to find Maximum Likelihood Estimates (MLE), repeating two steps until convergence.

**E-Step (Expectation):** Calculates each point's probability, or responsibility, of belonging to each cluster.

**M-Step (Maximization):** Updates cluster parameters to maximize data likelihood.

## The Math Behind It

GMM uses Mahalanobis distance, which measures distance while accounting for each cluster's covariance, so stretched or rotated clusters are handled better than with Euclidean distance.

## Cluster Parameters

**Mean ($\mu_k$):** The center of the Gaussian.

**Covariance Matrix ($\Sigma_k$):** The shape, spread, and orientation.

**Mixing Weight ($\pi_k$):** The prior probability, allowing clusters of different sizes.

## Hyperparameters (Covariance Types)

The `covariance_type` controls the allowed cluster geometry:

**spherical:** Circular shapes with different radiuses.

**diag:** Elliptical shapes, but axes stay parallel to feature axes.

**tied:** All clusters share the same shape, size, and rotation.

**full:** Clusters can have any ellipsoidal shape, size, and rotation.

## Limitations

**Complexity and Overfitting:** Slower and more expensive than K-means. Full covariance in high dimensions with limited data has a high overfitting risk.

# Hierarchical Clustering (Agglomerative)

**Category:** Hierarchical Clustering

## How the Model Works (Step-by-Step)

The most common approach is **Agglomerative (Bottom-Up)**, which is an iterative, greedy process:

1. **Start:** Every single data point begins as its own individual cluster (e.g., $N$ points = $N$ clusters).
2. **Find:** The algorithm calculates the distances between all existing clusters and finds the two closest ones.
3. **Merge:** It merges those two closest clusters into a single new cluster.
4. **Update:** It recalculates the distances between this new, larger cluster and all the other remaining clusters.
5. **Repeat:** It repeats Steps 2-4 until all data points have been merged into one single, massive cluster.

## The Math Behind It

The algorithm relies on two distinct levels of distance calculation:
1. **Distance Metric (Point-to-Point):** How the distance between individual data points is measured (e.g., Euclidean or Manhattan for quantitative data, Hamming for qualitative). Like K-Means, this requires feature scaling.
2. **Linkage Criteria (Cluster-to-Cluster):** Once points are grouped, this defines how the distance between *entire clusters* is measured to decide which two clusters to merge next.

## Hyperparameters (Linkage Criteria)

* **Single Linkage:** Minimum distance between any two points in different clusters. Prone to the "Chaining Effect" (long, thin, connected clusters).
* **Complete Linkage:** Maximum distance between points in different clusters. Encourages compact, spherical clusters but is highly sensitive to outliers.
* **Average Linkage:** Mean distance between all points in different clusters. A balanced approach that is less affected by outliers.
* **Ward's Method:** Minimizes the increase in within-cluster variance when merging. Very similar to K-Means inertia, but applied hierarchically. This is often the optimal choice in practice.

### Visual Comparison: Single vs. Complete Linkage

| Metric | Rule | Resulting Shape | Major Weakness |
| :--- | :--- | :--- | :--- |
| **Single Linkage** | **Min distance** between the two *closest* points. | Flexible, elongated (can follow "moons"). | **Chaining Effect:** Merges distinct groups if a single "bridge" point exists. |
| **Complete Linkage** | **Max distance** between the two *farthest* points. | Compact, tight, and spherical. | **Sensitivity to Outliers:** A single outlier can keep logical clusters apart. |

## Determining $K$ (Dendrograms)

Unlike K-Means, the number of clusters ($K$) is chosen **after** the model runs by analyzing a **Dendrogram** (a tree-like diagram recording the sequence of merges).
*   **Y-axis:** Represents the distance (or variance) at which clusters were merged. Larger vertical jumps mean the merged clusters were less similar.
*   **Cutting the Tree:** To determine $K$, draw a horizontal threshold line across the dendrogram. The number of vertical lines it intersects equals $K$.
*   **Best Practice:** Draw the cut across the largest vertical distance that does not intersect any horizontal merge lines. This yields the most distinct, natural clusters.

## Limitations

**Computationally Expensive:** The time complexity is high ($O(n^2)$ or $O(n^3)$). It cannot scale to massive datasets with millions of rows.
**Greedy Algorithm:** Once a merge is performed, it cannot be undone or re-evaluated later.
**Sensitivity to Outliers:** Depending on the linkage method used (especially Complete Linkage), outliers can severely impact the cluster structures.

# DBSCAN (Density-Based Spatial Clustering)

**Category:** Density-Based Clustering

**Video:** [DBSCAN Explained](https://www.youtube.com/watch?v=RDZUdRSDOok)

## How the Model Works (Step-by-Step)

Unlike centroid models that force data into rigid spheres, DBSCAN finds contiguous, arbitrary regions of high density:

1. **Pick:** The algorithm picks an arbitrary, unvisited point and evaluates the density within its **Epsilon ($\epsilon$)** radius.
2. **Classify:**
    * **Core Point:** If it has $\ge$ **MinPts** neighbors, a new cluster is started.
    * **Border Point:** If it has few neighbors but is within the radius of a Core Point, it joins that cluster.
    * **Noise Point:** If it is neither, it is flagged as an outlier (Noise).
3. **Expand:** If a point is a Core Point, all neighbors in its radius are added to the cluster. If those neighbors are *also* Core Points, their own neighborhoods are appended to the cluster (a chain reaction).
4. **Repeat:** The process continues until the cluster is completely surrounded by border points and noise, then it moves to the next unvisited point.

## The Logic: Point Classification

* **Core Points:** The "dense heart" of the cluster. They trigger the expansion.
* **Border Points:** The "edges" of the cluster. They are part of the group but don't have enough density to expand it further.
* **Noise Points:** Isolated anomalies that belong to no cluster.

## Hyperparameters

DBSCAN does not require a predefined $K$, but it is extremely sensitive to:

* **Epsilon ($\epsilon$ / eps):** The maximum radius used to search for neighbors.
    * **Tuning:** Use a **K-distance graph**; the "elbow" or inflection point indicates the optimal $\epsilon$.
* **MinPts (min_samples):** The minimum points needed to form a dense cluster.
    * **Rule of Thumb:** Typically set to $n_{dimensions} + 1$ (e.g., 4 or 5 for 2D data).

## Limitations & HDBSCAN

* **Varying Densities:** Standard DBSCAN fails if clusters have different densities, as one fixed $\epsilon$ cannot capture both.
* **The HDBSCAN Upgrade:** Hierarchical DBSCAN (State-of-the-Art) solves this by evaluating all possible $\epsilon$ values simultaneously to find stable clusters.
* **Curse of Dimensionality:** Accuracy degrades in high-dimensional space because Euclidean distance becomes less meaningful.

# Summary Decision Matrix: Clustering

| Model | Category | Scalability | Strengths | Weaknesses |
| :--- | :--- | :--- | :--- | :--- |
| **K-Means** | Centroid | **High** $O(nkdi)$ | Fast, simple, scalable. | Needs $K$, spherical bias, sensitive to outliers. |
| **GMM** | Distribution | **Moderate** $O(nkdi)$ | Flexible shapes (ellipses), soft assignment. | Needs $K$, sensitive to initialization, slow convergence. |
| **Hierarchical** | Hierarchy | **Low** $O(n^2)$ | Interpretable (Dendrogram), no predefined $K$. | Slow on large data, greedy (cannot undo merges). |
| **DBSCAN** | Density | **Moderate** $O(n \log n)$ | Complex shapes, noise detection, no $K$. | Sensitive to $\epsilon$ and density variations. |

# Metrics

## Inertia (WCSS) and the Elbow Plot

Inertia measures total within-cluster variance. It decreases as $K$ increases.

**Elbow Plot:** Choose the $K$ where inertia stops dropping sharply.

## The Silhouette Score

Scores clusters from -1 to 1; closer to 1 is better.

Measures cohesion within clusters and separation between clusters. Negative scores suggest poor assignment.

# PART 2: DIMENSIONALITY REDUCTION

## The Curse of Dimensionality

As the number of features (dimensions) in a dataset increases, the volume of the feature space grows exponentially. This causes several issues:
* **Data Sparsity:** Data points become isolated, making distance metrics (like Euclidean distance) lose their meaning.
* **Overfitting:** Models become too complex, increasing computational cost and the risk of memorizing noise.
* **Multicollinearity:** Features are highly correlated, destabilizing predictive models.

To solve this, we use Feature Extraction / Dimensionality Reduction to create fewer new variables that capture the essential information.

## 1. Principal Component Analysis (PCA) - *Linear*

**Category:** Linear Dimensionality Reduction

**Goal:** Transform a large set of correlated variables into a smaller set of *uncorrelated* variables (Principal Components) while retaining as much of the original variance (information) as possible.

### How it Works (The Math)
1. **Covariance:** Analyzes how all features move together.
2. **Eigenvectors (Directions):** Finds new orthogonal axes. The first axis (**PC1**) is drawn in the direction that maximizes the spread (variance) of the data. PC2 is drawn orthogonal to PC1, capturing the second most variance, and so on.
3. **Eigenvalues (Magnitude):** Gives a score to each new axis indicating how much total variance it explains.
4. **The SVD Trick:** In practice (e.g., Scikit-Learn), PCA uses **Singular Value Decomposition (SVD)** to compute these components instantly without needing the computationally heavy covariance matrix.

### Crucial Step: Preprocessing
**Standardization is Mandatory:** PCA is a variance-maximizing exercise. If Feature A is in millions (e.g., Revenue) and Feature B is in decimals (e.g., Conversion Rate), PCA will falsely think Feature A is more important just because its numbers are bigger. **Always apply a `StandardScaler` before fitting PCA.**

### Interpretation & Visualizations
* **Scree Plot (Choosing Components):** A line plot showing the percentage of variance explained by each principal component.
    * **Elbow Method:** Look for a steep drop followed by a plateau (the "elbow") to find the optimal number of components.
    * **Cumulative Variance Rule:** In business contexts, we often keep enough components to explain 70% to 80% of the total variance.
* **Correlation Circle:** Visualizes how original variables relate to the new Principal Components.
    * Arrows close together = positively correlated. Opposite = negatively correlated. Orthogonal (90°) = uncorrelated.
    * The closer an arrow is to the circle's border, the better it is represented by that 2D plane.
* **Biplot:** Combines the Correlation Circle (variables/arrows) and Sample Projections (data points/dots) on the same graph. This allows you to see *why* certain data points cluster together based on the original variables.

### Business Applications
1. **Creating Composite KPIs:** Synthesizing 15 complex metrics into a single "Global Score" (PC1).
2. **Fighting Multicollinearity:** Feeding strictly orthogonal (0 correlation) Principal Components into Regression models instead of highly correlated raw data, making the model highly stable.
3. **Preprocessing for Clustering:** Reducing dimensions (e.g., 200 features $\rightarrow$ 10 PCs) before running K-Means to defeat the curse of dimensionality and remove noise.
4. Anomaly/Fraud Detection: If a transaction projected onto PCA space lands extremely far from the center (origin), it violates standard correlation rules and is likely anomalous.

### Major Limitation
**Linear Hypothesis:** PCA has a major flaw: it assumes all relationships are linear. If your data lies on a complex, twisted, non-linear manifold (like a "Swiss Roll"), PCA will fail to capture the true structure.

## 2. Manifold Learning - *Non-Linear*

**Goal:** Beyond linear projections like PCA, these models use "Manifolds" to project non-linear, complex shapes (like a "Swiss Roll") into lower-dimensional space.

### t-SNE (t-Distributed Stochastic Neighbor Embedding)
**Category:** Non-Linear / Local Proximity Mapping

*   **Core Objective:** Focuses strictly on preserving **Local Neighborhoods**. It calculates the probability that two points are neighbors in high-dimensional space and attempts to match that exact probability in the 2D/3D projection.
*   **The "Crowding Problem" (Dimensionality Collapse):** High-dimensional space has significantly more "volume" than 2D. When squashing data into two dimensions, points that are moderately distant in high-dim space naturally get crushed together, creating overlapping blobs.
*   **The Solution (Heavy-Tailed t-Distribution):** t-SNE uses a Student’s t-distribution in the projection space. Because it has **heavy tails**, it exerts a stronger repulsive force on distant points, pushing clusters apart to "let them breathe" and making the gaps between them visible.
*   **Perplexity:** The primary tuning parameter that defines the number of effective neighbors each point considers.
    *   **Low Perplexity:** Prioritizes micro-clusters and fine-grained local patterns.
    *   **High Perplexity:** Captures the broader, macroscopic structure of the data.
*   **Critical Interpretation Caveats:**
    *   **Global Distances are Unreliable:** The physical gap between Cluster A and Cluster B on your screen does not accurately represent their global similarity.
    *   **Non-Deterministic:** Because the algorithm uses random initialization, clusters may appear in different positions or orientations in every run.
    *   **Density is Not Preserved:** A "dense" cluster in 2D doesn't necessarily mean the points were more tightly packed in high-dim space.

### UMAP (Uniform Manifold Approximation and Projection)
**Category:** Non-Linear / Topological Manifold Learning

*   **Core Objective:** Focuses on **Topological Structure Preservation**. Instead of just looking at neighbor probabilities, it builds a mathematical "skeleton" (a graph) of the entire dataset. It assumes the data is uniformly distributed on a surface (manifold) and tries to "unroll" that surface into a lower-dimensional space.
*   **The Global Advantage:** While t-SNE often "tears" the data apart to highlight local clusters, UMAP maintains the relative distance between those clusters. This means the macroscopic, **global geometry** of your data is preserved (e.g., if Cluster A is conceptually "closer" to Cluster B than to Cluster C, UMAP will likely show that).
*   **Performance & Versatility:** 
    *   **High Speed:** Significantly more computationally efficient than t-SNE; it can process millions of rows without massive downsampling.
    *   **d-Dimensionality:** Unlike t-SNE (which is mostly for 2D/3D visualization), UMAP can reduce data to *any* number of dimensions, making it a powerful pre-processing step for other machine learning models.
*   **Hyperparameters:**
    *   **n_neighbors:** Controls the "zoom level" of the topological graph. 
        *   Low values focus on fine-grained, local structures (individual trees). 
        *   High values capture the broader, global shape (the whole forest).
    *   **min_dist:** Controls how tightly the algorithm packs points together in the projection. 
        *   Low values create dense, clumpy clusters (useful for finding distinct groups).
        *   High values allow points to spread out (useful for seeing the continuity and "flow" of the data).

# Summary Decision Matrix: Dimensionality Reduction

| Model | Approach | Scalability | Main Goal | Best For |
| :--- | :--- | :--- | :--- | :--- |
| **PCA** | Linear (Global) | **High** | Variance Maximization | Feature reduction, Composite KPIs, Multi-collinearity. |
| **t-SNE** | Non-Linear (Local) | **Low** | Cluster Separability | **2D/3D Visualization only.** |
| **UMAP** | Non-Linear (Topological)| **High** | Structure Preservation | Fast reduction for big data, Pre-processing for clustering. |

# PART 3: ANOMALY DETECTION

## The Business Context & The True Cost
Anomaly detection focuses on finding the "needle in the haystack" (often <1% of the data). Because these events are so rare, standard classification accuracy is useless. Instead, models are tuned based on asymmetric financial risk:
*   **The Cost of a False Positive (FP):** The model flags a normal event. *Impact:* Wasted investigation time, "Alert Fatigue," or blocking a legitimate customer's credit card (Churn risk).
*   **The Cost of a False Negative (FN):** The model misses a true anomaly. *Impact:* A machine breaks down halting production ($100k/hour), or a massive fraud goes undetected.
*   **Contamination Parameter:** The threshold we set to balance FP and FN based on which risk is more dangerous to the business.

## The Two Paradigms & The Masking Effect
1. **Outlier Detection (The Messy Reality):** Unsupervised. The training dataset is "polluted" with unknown anomalies. The goal is to fit the central mode of the data and isolate far-away points.
2. **Novelty Detection (The Ideal Case):** Semi-supervised. Training is done on strictly *clean/normal* data (e.g., 100 hours of a perfectly healthy jet engine). The model learns this exact state and flags any future deviation.
*   **The Masking Effect:** Why not just use standard math (like Mean and Covariance)? Because outliers have a massive gravitational pull. They pull the calculated "center" towards themselves, masking their own anomalous nature and causing False Negatives. This requires robust algorithms.

## Key Models

### 1. One-Class SVM (Boundary-Based)
**Category:** Geometric Envelope / Novelty Detection
*   **Intuition (The Shrink-Wrap):** Imagine wrapping a tight plastic shrink-wrap around your dense, normal data. Anything falling outside this plastic boundary is flagged as an anomaly.
*   **The Mechanism:** Uses the **RBF Kernel** to project data into infinite-dimensional space, allowing it to draw complex, non-linear contours that tightly hug the normal data, no matter its shape.
*   **Hyperparameter - $\nu$ (Nu):** Represents your "margin of tolerance" for outliers in the training set.
    *   **Low $\nu$ (e.g., 0.01):** Creates a wide, trusting boundary. Assumes your training data is 99% clean.
    *   **High $\nu$ (e.g., 0.10):** Creates a tighter boundary. Actively excludes the outer 10% of your training data to prevent overfitting to noise.

### 2. Isolation Forest (Isolation-Based) - *SOTA for Big Data*
**Category:** Random Partitioning / Outlier Detection
*   **Intuition (A Paradigm Shift):** Instead of trying to define what is "normal," iForest actively targets the anomalies. Because anomalies are **"Few and Different"**, they are much easier to isolate from the rest of the pack.
*   **The Mechanism (Random Cuts):** The algorithm drops random vertical and horizontal "cuts" (splits) across the data.
    *   **Normal Points:** Are buried deep inside dense clusters. It takes *many* random cuts to finally isolate a single normal point. (Long Path Length).
    *   **Anomalies:** Sit out in empty space. It usually only takes 1 or 2 random cuts to isolate them. (Short Path Length).
*   **Strengths:** Extremely fast ($O(n \log n)$), handles high-dimensional data beautifully, and doesn't rely on complex distance calculations, making it the industry standard for massive datasets (like Web traffic or Logs).

### 3. Local Outlier Factor (LOF) (Proximity-Based)
**Category:** Density Comparison / Contextual Anomalies
*   **Intuition:** Global models (like iForest) often miss "Local" anomalies—points that look normal globally, but are anomalous relative to their specific, dense local neighborhood.
*   **The Mechanism:** LOF compares the density of a point $A$ to the density of its $k$ nearest neighbors.
    *   If $A$'s density is similar to its neighbors $\rightarrow$ $LOF \approx 1$ (**Normal**).
    *   If the neighbors are tightly packed, but $A$ is sitting slightly far away $\rightarrow$ $LOF \gg 1$ (**Anomaly**).
*   **Best for:** Contextual fraud and geospatial anomalies where local context is everything.

# Summary Decision Matrix: Anomaly Detection

| Model | Core Mechanism | Best Business Fit | Scalability |
| :--- | :--- | :--- | :--- |
| **Isolation Forest** | Random Splits | Massive logs, Bot detection, Web traffic | **Excellent** $O(n \log n)$ |
| **One-Class SVM** | Bounding Envelope (RBF) | Predictive Maintenance (Clean training data) | **Poor** $O(n^2)$ |
| **LOF** | Density Comparison | Contextual fraud, Local anomalies | **Moderate to Poor** |
